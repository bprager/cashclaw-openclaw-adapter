# `.codex` Workspace Guide

This folder keeps the minimum shared context needed to move fast without losing the thread.

## Recommended reading order
1. `PROJECT.md`
2. `STATUS.md`
3. `ARCHITECTURE.md`
4. `BACKLOG.md`
5. `DECISIONS.md`
6. `WORKLOG.md`

## What each file is for
- `PROJECT.md`: current repo reality, goals, constraints, and key paths
- `STATUS.md`: current progress snapshot, blockers, and next milestone
- `ARCHITECTURE.md`: intended system shape and integration boundaries
- `BACKLOG.md`: prioritized next work with a definition of done
- `DECISIONS.md`: durable choices and open questions
- `WORKLOG.md`: short running notes for future sessions

## Working style
- Prefer small, testable steps over broad rewrites.
- Verify the real repo state before trusting planning docs.
- Refresh `STATUS.md` when project state meaningfully changes.
- Record durable decisions in `DECISIONS.md`.
- Record session-level progress and blockers in `WORKLOG.md`.
- Keep these files concise so they stay worth reading.
